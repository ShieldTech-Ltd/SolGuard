# Security Policy

SolGuard is currently a hackathon prototype and must not be used to protect production funds.

## Reporting a vulnerability

Do not disclose a suspected vulnerability through a public GitHub issue. Use GitHub's private vulnerability reporting feature for this repository when available, or contact the maintainer privately through the contact method listed on the GitHub profile.

Include the affected component, reproduction steps, impact, and any suggested mitigation. Do not include real private keys, wallet seed phrases, access tokens, or personal information.

## Supported versions

No production version is currently supported. Security status will be documented after the first tagged release.

## Security posture

The repository documentation describes intended controls. A control is not considered implemented until code and automated tests exist in the repository.
